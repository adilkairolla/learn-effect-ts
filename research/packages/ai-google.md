## What it does

`@effect/ai-google` is the Google Gemini provider for the `@effect/ai` abstraction layer. It translates the provider-agnostic `LanguageModel.LanguageModel` interface into calls against `https://generativelanguage.googleapis.com/v1beta`, handling single-shot generation and SSE streaming. Without it, consumers hand-roll HTTP wiring, SSE parsing, JSON-Schema translation, and finish-reason mapping for every Gemini quirk (Gemma system-instruction injection, `thought`/`thoughtSignature` propagation, provider-executed code-execution round-trips).

## Public API surface

Five top-level modules, re-exported flat from `src/index.ts` (`repos/effect/packages/ai/google/src/index.ts:1-25`):

- **`Generated`** — auto-generated `S.Class` types from the Gemini OpenAPI discovery document: `GenerateContentRequest`, `GenerateContentResponse`, `Content`, `Part`, `SafetySetting`, `SafetyRating`, `GroundingMetadata`, `UsageMetadata`, `ExecutableCode`, `CodeExecutionResult`, `FunctionCallingConfig`, `UrlContextMetadata`. The `make` factory (`repos/effect/packages/ai/google/src/Generated.ts:4700-4729`) returns a typed `Client`. Regenerated by `scripts/generate.sh` via live OpenAPI fetch + `openapi-gen` + `scripts/generated.patch`.

- **`GoogleClient`** — `Context.Tag` wrapping the generated `Client` plus `generateContent` (maps errors to `AiError`, `repos/effect/packages/ai/google/src/GoogleClient.ts:138-165`) and `generateContentStream` (SSE via `Sse.makeChannel()`, terminates on first `finishReason`, `repos/effect/packages/ai/google/src/GoogleClient.ts:167-177`). Three construction paths: `make`, `layer`, and `layerConfig` (reads `apiKey`/`apiUrl` from `Config`).

- **`GoogleConfig`** — lightweight per-request `Context.Tag` carrying an optional `transformClient` function (`repos/effect/packages/ai/google/src/GoogleConfig.ts:13-24`). `withClientTransform` is a `dual` helper that injects HTTP middleware at call-site without rebuilding the client layer (`repos/effect/packages/ai/google/src/GoogleConfig.ts:43-56`).

- **`GoogleLanguageModel`** — the main integration point. `model(name, config?)` returns `AiModel.Model<"google", LanguageModel.LanguageModel, GoogleClient>` (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:169-173`). `Config.Service` is `Partial<Omit<GenerateContentRequest.Encoded, "contents" | "tools" | ...>>` so callers pass `safetySettings` or `generationConfig` directly as typed fields (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:59-76`). Module augmentation on `@effect/ai/Prompt` and `@effect/ai/Response` threads a `google.thoughtSignature` field through every part type (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:82-159`).

- **`GoogleTool`** — four provider-defined tool constants built with `Tool.providerDefined`: `CodeExecution`, `GoogleSearch`, `GoogleSearchRetrieval`, and `UrlContext`, with schemas taken directly from `Generated` types (`repos/effect/packages/ai/google/src/GoogleTool.ts:11-60`).

## Patterns used

- [Schema.Class and Schema.TaggedClass](../02-patterns-catalog.md#schemaclass-and-schemataggedclass) — every wire type in `Generated.ts` is an `S.Class` with JSDoc from the OpenAPI spec, giving both runtime codec and static type from a single declaration (`repos/effect/packages/ai/google/src/Generated.ts:29-45`).

- [Layer.succeed / effect / scoped — Layer constructors](../02-patterns-catalog.md#layersucceed--effect--scoped--layer-constructors) — `GoogleClient.layer` and `GoogleLanguageModel.layer` both use `Layer.scoped` to bind a `Scope` to the underlying HTTP connection lifecycle (`repos/effect/packages/ai/google/src/GoogleClient.ts:192-214`).

- [Config.string / integer / boolean / nested / all](../02-patterns-catalog.md#configstring--integer--boolean--nested--all) — `layerConfig` collects `apiKey`/`apiUrl` through `Config.all`, making the client composable with any `ConfigProvider` (`repos/effect/packages/ai/google/src/GoogleClient.ts:220-246`).

- [Context.GenericTag / Tag class / Reference — tag variants](../02-patterns-catalog.md#contextgenerictag--tag-class--reference--tag-variants) — `GoogleClient`, `GoogleConfig`, and `GoogleLanguageModel.Config` all use the class-based `Context.Tag` idiom (`repos/effect/packages/ai/google/src/GoogleClient.ts:31-33`, `repos/effect/packages/ai/google/src/GoogleConfig.ts:13-24`, `repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:38-49`).

- [Dual data-first / data-last (dual(...)) and Pipeable trait](../02-patterns-catalog.md#dual-data-first--data-last-dual-and-pipeable-trait) — `GoogleConfig.withClientTransform` is defined with `dual(2, ...)` so it works both as a standalone function and as a pipeable operator (`repos/effect/packages/ai/google/src/GoogleConfig.ts:43-56`).

- [Stream.make / fromIterable / fromEffect](../02-patterns-catalog.md#streammake--fromiterable--fromeffect) — `makeStreamResponse` maps each SSE chunk to a `StreamPartEncoded[]` array, then `Stream.flattenIterables` unrolls them without materialising the full response (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:583-767`).

- [Redacted — prevent secret values from leaking to logs/spans](../02-patterns-catalog.md#redacted--prevent-secret-values-from-leaking-to-logsspans) — the API key is typed as `Redacted.Redacted`; `Effect.locallyScopedWith(Headers.currentRedactedNames, Arr.append("x-goog-api-key"))` ensures it is also scrubbed from HTTP diagnostic logs (`repos/effect/packages/ai/google/src/GoogleClient.ts:80-82`).

- [The internal/ folder and index.ts re-export shape](../02-patterns-catalog.md#the-internal-folder-and-indexts-re-export-shape) — `src/internal/utilities.ts` holds JSON-Schema-to-OpenAPI conversion and `finishReason` mapping; excluded from public exports via `"./internal/*": null` (`repos/effect/packages/ai/google/package.json:37`).

## What's unique about this package's design

The most instructive decision is streaming without a separate DTO. `generateContentStream` builds the SSE URL inline, adds `alt=sse`, and pipes through `Sse.makeChannel()`, reusing the same `Generated.GenerateContentResponse` Schema for each event (`repos/effect/packages/ai/google/src/GoogleClient.ts:167-178`). One codec serves both the single-shot JSON path and the per-chunk SSE path.

A second Gemini-specific necessity is the Gemma workaround: when the model name starts with `gemma-`, the system prompt is injected as a text prefix in the first user message rather than using the `systemInstruction` field, which Gemma ignores (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:405-413`).

Provider-executed tools appear as `tool-call`/`tool-result` pairs with `providerExecuted: true`, keeping them in conversation history without requiring a user handler loop (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:497-529`). A `Set<string>` (`emittedSourceUrls`) deduplicates grounding sources across SSE chunks (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:581`).

## Conventions observed

- **Generated.ts is a code-gen artefact.** `scripts/generate.sh` fetches the live Gemini OpenAPI discovery doc, runs `@tim-smart/openapi-gen`, and applies `scripts/generated.patch`. The file is never hand-edited; treat it as vendored code.
- **Exhaustive error mapping at I/O boundaries.** `catchTags({ RequestError, ResponseError, ParseError })` converts all platform errors to `AiError` subtypes (`repos/effect/packages/ai/google/src/GoogleClient.ts:115-135`, `143-163`).
- **Context merging at call time, not layer-build time.** Config overrides are read via `context.unsafeMap.get(Config.key)` inside `makeRequest` so callers can inject them with `Effect.provideService` without rebuilding layers (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:188`).

## "If you were authoring something similar, copy this"

- **One Schema class, two transports.** Reuse the same `S.Class` for both the JSON endpoint and the SSE stream — same codec, no separate streaming DTO (`repos/effect/packages/ai/google/src/GoogleClient.ts:167-178`).
- **`Effect.locallyScopedWith` for header redaction.** Register sensitive header names into `Headers.currentRedactedNames` inside the scoped `make` so they unregister automatically on scope close (`repos/effect/packages/ai/google/src/GoogleClient.ts:82`).
- **Module augmentation for provider metadata.** `declare module "@effect/ai/Response"` to attach `google?: { thoughtSignature? }` on every part type — provider data travels with generic types without breaking the shared contract (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:82-159`).
- **`providerExecuted: true` flag.** Mark server-side tool results so callers distinguish model-executed actions from user-executed ones without inspecting provider-specific fields (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:506`).

## Open questions

- `scripts/generated.patch` is applied post-generation but undocumented: unclear which fields it fixes, and an upstream OpenAPI spec change could silently break or mis-apply it.
- `jsonSchemaToOpenApiSchema` throws for `$ref` schemas ("Reference resolution not implemented") (`repos/effect/packages/ai/google/src/internal/utilities.ts:96-100`). Recursive or cross-referencing tool schemas will fail at runtime with no documented workaround.
- `prepareTools` rejects mixing provider-defined and user-defined tools in one request (`repos/effect/packages/ai/google/src/GoogleLanguageModel.ts:846-853`). Whether this is a Gemini API constraint or a simplifiable limitation is unconfirmed.
- The `cachedContent` field on `GenerateContentRequest` enables Gemini context caching but has no dedicated helper — callers must use the raw `Config.Service` escape hatch.
